## v2.8.0 — 2026-07-24 · 管理分析と運用

### 新機能
- **管理分析ダッシュボード** — ストレージ、トラフィック、ユーザー、共有、
  リモートダウンロード、バックグラウンドジョブを対象としたライブ運用カードと
  信頼性の高い時間単位ロールアップ。トレンド、カバレッジ表示、バックフィル
  ツールも備えます。
- **ストレージ運用** — プロバイダープリセット、接続テスト、リクエストプレビュー、
  カテゴリ別使用量予測、ファイル配置追跡、安全なクリーンアップを備えた管理画面。
- **スペースとクォータ** — ワークスペース設定をメンバー、サービス、請求、
  画像ホスティングに整理し、クォータ所有権とスペース間操作を明確化。
- **公開プロフィールと共有** — 公開ページを刷新し、厳選した共有をユーザーの
  プロフィールへ直接掲載できます。
- **WebDAV 管理** — サービス制御と任意のカスタムドメイン、自動導出・検証に対応。
- **API とイベント** — Scalar ドキュメント付きの完全な OpenAPI、統一された
  リソーススキーマとエラー、フロントエンドのポーリングを置き換える SSE ストリーム。
- **その他の改善** — 管理者向けユーザー/チーム活動、監査フィルター、
  リモートダウンロード履歴、NFO プレビュー、メール認証設定、前回のログイン方法の記憶。

### 修正
- リモートダウンローダーの監視、heartbeat 復旧、シード、再試行、
  クレジット事前承認、進捗報告、クリーンアップを全面的に改善。
- 分析は範囲が確定した完了済みロールアップのみを使用し、不完全な履歴を
  部分データやライブデータと混在させず明示するようになりました。
- 非 ASCII ファイル名のアップロードメタデータと、アップロード後の保存済み
  Content-Type 同期を修正。
- Finder の WebDAV 互換性、使用中フォルダの保護、ダイアログのオーバーフロー、
  フォルダツリーの折りたたみを改善。

> **破壊的変更：** API のパスとレスポンスをリソース指向に統一しました。
> エラー形式も統一され、削除は `204 No Content` を返し、状態遷移には
> リソースの status エンドポイントを使用します。ランディングページ形式の共有は、
> 明示的に非公開へ設定しない限り、デフォルトで公開アクセス可能になりました。

[Full release notes ↗](https://github.com/saltbo/zpan/releases/tag/v2.8.0)

## v2.7.0 — 2026-06-06 · リモートダウンロード、WebDAV ほか

### 新機能
- **リモートダウンロードマネージャー** — torrent/HTTP ダウンロードをリモートワーカーへオフロードし、
  詳細なタスクインスペクター、ピアの地理的リージョン、BT のシード保持、
  ドライブへフォルダ構造を保ったままアップロードする機能を提供。
- **`zpan` CLI ダウンローダー** — ワンコマンドのデバイスログインと設定可能なサーバー URL。
- **WebDAV アクセス** — ユーザーごとのアプリパスワードで WebDAV 経由でドライブをマウント
  （RFC 4918 Class 2 互換）。
- **サーバーサイドアーカイブ** — ストリーミング ZIP ジョブをキューに入れ、新しい
  バックグラウンドタスクページで追跡。
- ウェブ UI での**フォルダアップロード**。
- **クラウドクレジット** — クレジットで課金される従量制のストレージ送信、クレジットストア付き。
- サインインとサインアップ向けの **Captcha** 保護。
- 統一された API キー管理。

### 修正
- リモートダウンロードのライフサイクル（リセット、リカバリー、シード処理）を堅牢化し、
  プレビューとアップロードに関する各種修正を実施。

> **破壊的変更：** より厳格な RESTful API ルート。公開ダウンロードリンクが
> `/dl/:token` から `/r/:token` へ移動。
