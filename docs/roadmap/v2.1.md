# v2.1 — Image Bed

Turn ZPan into a proper image bed with tool ecosystem integration.

## Major Features

### Upload API
- REST API for programmatic file upload (token-authenticated)
- API returns configurable URL formats: raw URL, Markdown `![](url)`, HTML `<img>`, BBCode

### Tool Integrations
- **PicGo / PicList compatible endpoint** — works with Typora, Obsidian, and other Markdown editors
- **ShareX Custom Uploader** — `.sxcu` config file generator in the web UI, double-click to configure
- **Flameshot** compatible — same upload protocol as ShareX

### Upload Experience
- Clipboard paste upload in web UI (paste screenshot, get URL instantly)
- Batch upload with progress indicators
- Upload history panel — recently uploaded files with one-click URL copy
- Auto-copy URL to clipboard after upload (configurable format)

### Migrate raw SQL to Drizzle query builder
- All 48 `db.all<T>(sql\`...\`)` / `db.run(sql\`...\`)` calls in services/ and routes/ use hand-written SQL with manual type assertions, completely bypassing Drizzle's schema-based query builder
- Rewrite to use `db.select().from(matters).where(...)`, `db.insert(storages).values(...)`, etc.
- Key wins: schema changes trigger TypeScript errors in queries, column name typos caught at compile time, Drizzle relations/joins become usable, migration diffs are reliable
- Scope: `services/matter.ts` (24), `services/storage.ts` (7), `services/user.ts` (6), `routes/quotas.ts` (5), `routes/system.ts` (7)

### Custom File Manager (replace SVAR)
- Replace `@svar-ui/react-filemanager` with a custom file manager built on shadcn/ui primitives
- Motivation: SVAR has poor theming (clashes with shadcn), upload bypasses event bus (requires DOM hacking), path-based ID model doesn't fit database IDs, no headless mode, tiny community (51 GitHub stars)
- Stack: `@tanstack/react-table` (list view), shadcn `ContextMenu` / `Breadcrumb` / `ToggleGroup` (UI), `@dnd-kit` (drag & drop), existing `UploadDropzone` + `FilePreviewDialog` (upload & preview)
- Key wins: native presigned URL upload support, unified Tailwind theming, zero CSS hacks, full control over UX

## User Scenarios

**Blogger writing in Obsidian:**
> I paste a screenshot into Obsidian, PicGo uploads it to my ZPan, and inserts `![](https://img.myblog.com/a1b2c3.png)`. I never leave my editor.

**Screenshot workflow (ShareX user):**
> I press PrintScreen, ShareX captures my screen, uploads to ZPan in the background, and puts the URL in my clipboard. I paste it into Slack. Total time: 2 seconds.
